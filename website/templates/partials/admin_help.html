Editor's guide

Quick reference for what each page renders and where to edit it. Every link below opens the relevant Django admin screen in a new tab.

Home Profile Enterprise Leadership Case Studies Innovation Perspectives Connect Resume Footer & Navigation

Home

Path: /

  • Hero headline, positioning line, sub-headline — Site Config → Homepage hero.
  • Vision section (heading + body) — Site Config → "Homepage Vision section". Hide the section by clearing the body.
  • Enterprise column (heading, intro, closing) — Site Config → "Homepage Enterprise Leadership column".
  • Innovation column (heading, intro) — Site Config → "Homepage Innovation column".
  • Pillar bullets under each column — Homepage Pillars. Set Category to Enterprise or Innovation; order with Position.
  • "Read the executive profile →" link label — Site Config → "Read the executive profile" link label.

Profile

Path: /profile/

  • Page title, intro — Pages → choose profile.
  • Body sections — Page Blocks inline on the same screen. Each block has its own heading + body. Reorder with Position.

Enterprise Leadership

Path: /enterprise-leadership/

  • Hero heading + lead paragraph, Scope, Impact, "What I lead" heading, System Integration heading + body — Enterprise Overview.
  • The three function blocks (Compensation, HRIS, Payroll) — Enterprise Functions. Each function has Title, Summary, Responsibilities, Systems Led, Role in the Organization. Reorder with Position.
  • Eyebrow labels ("Scope", "Impact", "Responsibilities", etc.) are intentionally hardcoded — they label the structure, not the copy.

Case Studies

Path: /case-studies/ and /case-studies/<slug>/

  • Add or edit a case study — Case Studies. Each one has four structured fields: Problem, Role, Approach, Outcome. Keep them as separate fields — never collapse into one freeform body.
  • Toggle visibility with the Published checkbox.
  • Order on the index page — Position field.

Innovation

Path: /innovation/

  • Hero heading, intro, Beacon positioning, WLJ overview, architecture caption, AI Chief of Staff, "What this demonstrates", case-study link label — Innovation Overview.
  • The Data → Signals → Decisions diagram is rendered in SVG and is intentionally code-controlled. Change the caption underneath it via the Architecture caption field.

Perspectives

Path: /perspectives/ and /perspectives/<slug>/

  • Add or edit an essay — Perspectives.
  • Body sections are inline Perspective Sections — heading + body, ordered by Position.
  • Toggle visibility with Published.

Connect

Path: /connect/

  • Page intro, form prelude, success heading, success body — Connect Page.
  • Form field labels (Name, Email, Reason for reaching out, Message) are intentionally code-controlled.
  • Inquiry-type options come from the form definition; change them in code if the dropdown needs new categories.

Resume versions

Path: /resume/<slug>/

  • Each version — Resume Versions. Toggle Active to make a slug live.
  • Resume sections are inline (Experience, Education, etc.) with heading + body, ordered by Position.

Footer & navigation

  • Contact email and LinkedIn URL in the footer — Site Config → Footer & contact.
  • Top-nav labels and the "Danny R. Jenkins" brand wordmark are intentionally code-controlled.
  • Page <title> tags and meta descriptions live in templates. If you want one in admin, ask and we'll wire it.

Editing conventions

  • Headings live in CharFields, never inside the rich-text editor. The editor strips heading tags by design.
  • Singletons (Site Config, Enterprise Overview, Innovation Overview, Connect Page) always have a single record at id=1. Never create a second.
  • Production sync mode — when CONTENT_SYNC_MODE=run, deploys preserve admin edits. Default is build, which upserts from the fixture.
  • Static files — if a CSS or image change isn't appearing, the edit may need a fresh collectstatic on deploy.
